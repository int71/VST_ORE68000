・インストール方法
  ・ディレクトリ「VST_ORE68000.vst3」を「C:/Program Files/Common Files/VST3」配下にコピー
    「ショートカット」でもイケるようですが、その場合でも名前は「VST_ORE68000.vst3」にしてください。
  ・依存ライブラリの有効化
    下記のいづれかになります。
    上から順で推奨します。

    a. ライブラリが置かれているディレクトリを「パス」に追加
      手順は下記になります。
      ・エクスプローラ等の「PC」を右クリックして「プロパティ」

    b. 「パス」が通っているディレクトリ(C:/Windows/System32等)配下にコピー
    c. VSTホスト(Cubase等のDTMソフトウェア、VST3PluginTestHost等のテスト用ホスト等)実行ファイルディレクトリ配下にコピー
  ・ホストからの使用
    「VST_ORE68000」という名前で自動的に見つけてくれるはずです。

・VSTホスト
  打ち込み等の本気使いをするならCubase等のDTMソフトウェアが必須と思いますが、動く様を見るだけであれば「VST3PluginTestHost」を利用できます。
  ただし単体での再配布が許されているか不明なので、「https://www.steinberg.net/ja/developers/」からSDKをダウンロードし、インストールしてください。

・「YM2151」と「YM2151B」の違い
  「VST_ORE68000」はYAMAHA製FM音源「YM2151」をベースに独自拡張を行った架空の音源「YM2151B」を再現します。
  FM音源機能は「ほぼ」互換ですが、下記が拡張(非互換)となります。

  ・KeyOnノイズ軽減機能
    KeyOnした際のプチノイズを軽減します。
    ただし万能ではなく、音色によってはネガティブに働く可能性があります。
  ・左右チャンネル独立の音量設定機能
    ソフトシンセである以上ホスト側でどうとでもなる部分ではありますが、
    「YM2151B」仮想ハードウェアレベルで機能追加しています。
  ・PCM音源機能
    8bitモノラル音声(音量は左右チャンネル独立で設定可能)を再生できるPCM音源機能が追加されています。
    ゼロからの追加で、これが別称「B」を名乗る所以です。
    FM音源と独立した8チャンネルを持ち、波形メモリは64KBです。
    こちらもソフトシンセ利用者からは関係無い事ではありますが、
    音程、音量パラメータがFM音源互換であるので、仮想「YM2151B」を使用したソフトウェアは楽ができます。

・ソフトウェアシンセサイザーとしての仕様
  ・MIDI入力16チャンネル、音声出力2チャンネル
  ・MIDI入力チャンネル1〜8番はFM音源、9〜16番はPCM音源で固定

・UI思想
  ・音源種別、チャンネルはタブで切り替え
  ・下部キーボードはマウス操作が基本で、「Focus」スイッチON時は(文字入力する方の)キーボードでも操作可能
  ・「Polyphony」スイッチON時は和音発声可能
    ただし、発声がFM、PCMそれぞれ単一のチャンネルに制限されます。
  ・「チャンネルへの音色割り当て」と「編集対象の音色選択」を区別しない
  ・演奏中の音色切り替えはUIに反映される
    これにより、激しく音色が切り替わる曲だと音色編集も激しく困難になります。
  ・設定、音色、波形サンプル情報をそれぞれファイルに書き出せる
  ・PCM音源の波形情報は音色情報と独立して存在
    「音色毎に1つの波形を選択する」という設計です。

・(雑な)使用方法
  ・「Common」タブ
    ・Clock
      仮想「YM2151B」の動作クロックを設定します。
      音程は(ソフトシンセ内部での補正により)変化しませんが、LFO周波数、エンベロープ速度等に影響します。
      仮想「YM2151B」を4MHzで駆動するソフトウェア向け機能であるので、通常は使用しないでしょう。
    ・Denoise
      ON時、KeyOn時のノイズ軽減を行います。
      当該チャンネルが無音でない状態でKeyOnした場合に、音量、波形位相のギャップを軽減します。
      音色によっては発声開始時に問題が起きるかも知れません。
    ・LFO
      音量、音程を揺らすのに使用される、低周波発生機の設定です。
      音源全体で1つだけ存在し、16チャンネル全てで共有されます。
      ・Shape
        波形を設定します。
        「Delta」以外は使いにくいと思います。
      ・Freq.
        周期を設定します。
        数字が大きければ大きいほど高周波になります。
      ・AMD
        音量変化の深さを設定します。
      ・PMD
        音程変化の深さを設定します。
  ・「FM」タブ
    ・Polyphony
      ONにすると和音発声モードになります。
      ただし、発声がFM、PCMそれぞれ単一のチャンネルに制限されます。
    ・「Program(音色)」選択プルダウン
      チャンネルに設定する音色、もしくは編集対象の音色を指定します。
      音色名も変更できます。
      演奏中の曲で音色切り替えが発生すると、ここの表示も切り替わります。
    ・「Export Program Bank」、「Import Program Bank」
      「Program(音色)」データをファイルとして入出力します。
      「.xml」が「VST_ORE68000」のネイティブになりますが、「.opm」形式も読み書き出来ます。
    ・「チャンネル」属性のパラメータ
      ・Vol.
        音量を設定します。
        127が最大です。
      ・Pan
        左右定位を設定します。
        -63が左端、+63が右端です。
      ・AMS
        当該「チャンネル」の音量変化の深さを設定します。
        「Common」タブの「LFO/AMD」値に乗じられます。
      ・PMS
        当該「チャンネル」の音程変化の深さを設定します。
        「Common」タブの「LFO/PMD」値に乗じられます。
    ・「Program(音色)」属性のパラメータ
      「なり」に「YM2151」互換なので、先人のドキュメントをご参照ください。
      なお、「Copy」、「Paste」で1音色単位のコピペが出来ます。
      実体がテキストなので、テキストエディタ越しに編集する事もできます。
  ・「PCM」タブ
    ・Polyphony
      「FM」同様です。
    ・「Program(音色)」選択プルダウン
      「FM」同様です。
    ・「Export Program Bank」、「Import Program Bank」
      「FM」同様です。
    ・「チャンネル」属性のパラメータ
      「FM」同様です。
    ・「Program(音色)」属性のパラメータ
      「TL」〜「KS」は「YM2151」FMオペレータ同様の意味になります。
      オペレータ解釈だと「1オペレータ」なので、「AM」有効無効スイッチはありません。
      # 「AMS」を変えればいいから
    ・「Sample(波形)」属性のパラメータ
      「Sample(波形)」は「Program(音色)」と別に存在し、
      「Program(音色)」毎に1つの「Sample(波形)」を割り当てます。
      複数の「Program(音色)」から1つの「Sample(波形)」を割り当てる事ができ、
      同一波形で「TL」〜「KS」のパラメータを別にする、という(節約目的の)技が使えます。
      ・「Sample(波形)」選択プルダウン
        ここで名前も変更でき、アルファベット順にソートされます。
        名称重複時はカッコ数字が付加されます。
        初期状態では「Default」1つだけが存在します。
      ・Add
        「new sample」という名称で波形情報を追加します。
        「Usage(使用量)」が「65536-64=65472」以下(空き64以上)である必要があります。
      ・Remove
        波形情報を削除します。
        残り1波形である場合は何も起きません。
      ・Mode
        「Internal(波形メモリ音源再現モード)」と「External(外部音声ファイル使用モード)」を切り替えます。
      ・Interpolation
        サンプル補間モードを設定します。
        ・None
          補間無効とします。
          矩形波等はシャープに再生されますが、「高周波成分によるノイズ」になる場合があります。
        ・Linear
          線形補間します。
          ノイズは軽減されますが、波形周波数が低いと籠って聞こえる場合があります。
      ・Denoise+
        ON時、KeyOn時のノイズ軽減を行います。
        PCM音源は「位相」の概念が希薄なので、
        「Common」タブの「Denoise」スイッチをONにしただけだと音量の補正だけが行われますが、
        「Denoise+」をONにするとFM音源同様の「位相」補正も行われるようになります。
        ループ対象が波形全体であれば、「位相」相当の概念があると見なせます。
      ・「Mode」別インタフェイス
        ・Internal(波形メモリ音源再現モード)
          ・波形ウィンドウ
            マウス左クリックで波形を描きます。
          ・H Reso.
            波形サンプル数を設定します。
            16〜64を設定できます。
          ・V Reso.
            1サンプル解像度ビット数を設定します。
            が、小さくしたとしても波形メモリの節約にはなりません。
            飽くまで波形メモリ音源再現のための「お遊び」です。
          ・数値入力欄
            波形ウィンドウで描く波形のサンプル1つ1つを数値編集できます。
            「H Reso.」64時、ページ切り替えが発生します。
        ・External(外部音声ファイル使用モード)
          ・波形ウィンドウ
            ・表示波形
              ループ外再生波形は白(半透明なのでグレー)、ループ対象は緑、切り捨て部分は青で表示されます。
              緑、青の波形はループ前後に「ゴースト」としても表示され、ループ連続性を視覚的に確認できます。
            ・左クリック
              「Offset(再生開始位置)」、「Loop(ループ長)」、「Length(サンプル長)」を設定できます。
              設定対象のパラメータはそれぞれの数値入力欄クリックで切り替わり、「Edit:」の右に表示されます。
            ・右クリック
              ドラッグしてスクロールさせます。
            ・ホイール
              表示を拡大縮小します。
          ・AMP
            波形高さ(振幅)を拡大縮小します。
            毎回オリジナル波形から計算し直すので、小さい値から大きい値に変更しても劣化はありません。
          ・Load
            波形データを読み込みます。
            現状「.wav」だけが使用可能です。
          ・ファイル名、サンプル数表示
            読み込んだ音声のファイル名、サンプル数が表示されます。
            65536を超える値も表示はされますが、編集によりサンプル数縮小を行わない限り、全体を再生する事はできません。
          ・Octave、Note、Fine
            読み込んだ音声の基準音程を設定します。
            ここで設定した音程がMIDI入力された場合に、オリジナル音声と同じ音程で再生される事になります。
            Fineは「-128〜127」が指定可能で、1オクターブ12音階の±1/2に対応します。
          ・Clear
            読み込んだ音声を解放します。
          ・Frequency、サンプリング周波数変更後サンプル数表示
            サンプリング周波数を変更します。
            高ければ高いほど音質は良くなりますが、その分波形メモリを消費するため、
            通常「妥協できる最低ライン」を探る使い方になります。
            毎回オリジナル音声からサンプリングし直すため、低周波数から高周波数に更新しても音質劣化はありません。
            また、「Octave」、「Note」、「Fine」は内部調整されるため、
            「Frequency」変更の度に編集する必要はありません。
            後述の「Offset」、「Loop」、「Length」も自動調整されますが、
            こちらは調整が「やり切り」であるため、
            何度も変更すると「切り捨て」による誤差が発生する場合があります。
            サンプリング周波数変更後サンプル数表示もオリジナル音声サンプル数表示同様、
            65536を超える値も表示はされますが、編集によりサンプル数縮小を行わない限り、全体を再生する事はできません。
          ・Offset
            再生開始位置を指定します。
            これ以前の波形は波形メモリ上に載りません。
          ・Loop
            ループ長を指定します。
          ・Length
            サンプル長を指定します。
            この数値が波形メモリ消費量となります。
          ・Loop/Length Lock
            OFF時はループ開始、終了位置指定が独立になりますが、
            ONにすると、ループ開始、終了の間隔が固定されるようになります。
          ・Reset
            音声全体が再生対象になるよう、「Offset」、「Loop」、「Length」を設定し直します。
  ・鍵盤
    マウスで発声指定できる他、演奏データによる入力状況も表示されます。
    「Common」タブ選択時は、直前まで選択されていた音源に対する発声指定になります。
    ・Octave
      鍵盤左端「ド」のオクターブ番号を設定します。
    ・Focus
      (文字入力する方の)キーボードからの入力を(VSTホストへの伝達を横取りして)受け付けます。
      「Polyphony」がONであれば和音も入力できます。
      ただし数値入力にフォーカスが移った場合は横取りしません。
  ・「Export Settings」、「Import Settings」
    「VST_ORE68000」全体の設定情報を「.xml」で読み書きします。
    ここには音色、サンプル情報の他、UI状態も含まれます。

・動作確認
  「Cubase Version 13.0.41 Build 256(x64)」および「VST3Plugin TestHost Standalone - Version 3.8.0 Build 2」にて動作確認しています。
